# AI 产品经理 · 多场面试批量复盘

## 定位

与 `ai-pm-interview-review` 上下游关系：

```
单场问答 → ai-pm-interview-review → 单场复盘文档
                                          │
                              多场攒够 → 本 skill → 横评总结 + 题库
```

本 skill 的输入是**已有的单场复盘文档**，不处理原始面试对话。

## 工作流程

### 第一步：收集输入

**按优先级尝试：**

1. **从 VIP Docs 文件夹读取**（默认）：用户提供文件夹链接或 GUID，默认使用「个人复盘」`kgz48nDOYJyRfBdQ`。调用 vip-docs skill 的 `export_file` 逐文件导出。
2. **回退方案**：用户直接粘贴多篇复盘文档内容。

**提取每场元数据**：日期、公司、评分、各象限评级分布。

### 第二步：横评分析

读取 `references/cross-analysis-framework.md`，对全部场次执行四项分析：

1. **能力矩阵**：按四象限汇总各场表现，标注趋势（↑进步 / ↓退步 / →持平）
2. **高频失分模式**：跨场识别重复出现的失分类型，按出现频率 × 严重程度排序 TOP5
3. **能力变化趋势**：各象限多场走势，识别哪个象限持续弱、哪个在进步
4. **改进优先级**：按 ROI 排序（影响面 × 修复难度的倒数），输出 Top 3 行动项

### 第三步：生成输出文档

**文档 A：多场横评总结**（始终生成）
按 `references/batch-output-template.md` 的「文档A」结构：
- 五场概览表
- 四象限能力横评（每象限：共性优点 + 共性短板 + 代表性证据）
- 高频失分 TOP5（排名+出现场次+严重程度+简短改进方向）
- 能力变化趋势表
- 改进优先级 Top 3
- 一句话总结

**文档 B：题库与参考回答**（当用户明确要求时生成，或在生成总结后询问用户是否需要）
按 `references/batch-output-template.md` 的「文档B」结构：
- 象限一～四：全量问题列表（合并相似问题）
- 针对 🔴需改进 和 🟡合格 的问题，给出改进版回答
- 每道题标注：出场场次 + 原答问题 + 改进版回答

### 第四步：保存文档

1. **VIP Docs**（强制）：使用 vip-docs skill 的 `import_file` 上传到「个人复盘」文件夹（默认 `kgz48nDOYJyRfBdQ`），file_content base64 编码。
2. **本地副本**：同步保存到当前工作目录。

命名格式：
- 文档A：`面试横评总结-前N场.md`
- 文档B：`面试题库+参考回答-前N场.md`

### 第五步：概要汇报

3-5 行总结：整体评分区间、最强象限、最弱象限、最关键的一个改进方向。
