# 横评分析框架

## 一、能力矩阵分析

### 评分归一化

将各场的四象限评级归一化为五级量化：
- 🟢优秀 = 5
- 🟢→🟡 = 4
- 🟡合格 = 3
- 🟡→🔴 = 2
- 🔴需改进 = 1
- 无法评分 = 0（不纳入计算）

### 趋势判断

对比同一象限在多场中的评分变化：
- **↑进步**：评分逐场上升 ≥2 级
- **→持平**：评分波动在 1 级以内
- **↓退步**：评分逐场下降 ≥2 级
- **⚠波动**：评分大幅波动无方向性

### 横评输出结构

每个象限按以下结构组织：

```
### 象限X：[名称]（总体评价：评级）

**共性优点**：
- [优点1]：出场场次 + 代表性证据
- [优点2]：出场场次 + 代表性证据

**共性短板**：
- [短板1]：出场场次 + 代表性证据
- [短板2]：出场场次 + 代表性证据
```

## 二、高频失分模式识别

### 识别规则

跨场扫描所有 🔴需改进 标记，按相似根因聚类：

| 模式类型 | 聚类判据 |
|---|---|
| 技术深度不扛压 | 涉及「没考虑到」「技术方案说不清」等 |
| 项目收益缺量化 | 涉及「面试官追问才有数据」「答非所问」等 |
| 对齐姿态被动 | 涉及「嗯嗯」「面试官在教候选人」等 |
| 回答停留在方向层 | 涉及「面试官说没回答到点上」等 |
| 目标公司缺研究 | 涉及「动机放任何公司都能用」等 |

### 排序算法

```
优先级 = 出现场次 × 权重 + 严重程度 × 权重
```
- 出现场次权重：1
- 严重程度权重：2（🔴致命 = 3，🟡需改 = 1）

### 输出格式

| 排名 | 失分模式 | 出现场次 | 严重度 | 影响范围 | 改进方向 |
|---|---|---|---|---|---|
| 1 | XX | N/5 | 🔴致命 | Q3、Q4 | [一句话] |

## 三、改进优先级

### 排序算法

```
ROI = 影响面（涉及几个象限）× 修复难度倒数
```

- 影响面：该改进涉及几个评估象限（1-3）
- 修复难度倒数：1/估算所需小时数

### 输出格式

按 ROI 降序排列 Top 3：
1. **[行动项]**：涉及象限 + 预计耗时 + 一句话原因

## 四、特殊处理规则

### 合并相似问题

多场面试中内容高度相似的问题，在题库中合并为一条，标注出场场次。

合并判据：
- 问题意图相同（如「为什么换工作」「为什么看外部机会」→ 合并）
- 回答评估结论可互相印证或补充

### 回答缺失处理

若某场仅有面试官提问无候选人回答（评分 N-A），仅纳入问题列表和面试官间接信号分析，不纳入能力评分计算。

### 评分冲突处理

若同一候选人在不同场次同一象限表现差异大（≥2级），标注为「⚠波动」，并分析可能原因（面试官风格、题目难度、状态等）。
